# Historie
Diese Seite fasst die sichtbare Projekthistorie von `UnyPort` anhand des Repositories und der Release-Dateien in `docker_unyport` zusammen.

## 3. Mai 2026 - erste Dashboard- und Netzwerkbasis
Die ersten Commits vom `2026-05-03` legten die fruehe Produktbasis:

- Erste Dashboard-Arbeit
- Aufgewertete Präsentation
- Erste Netzwerk-Chart- und Datenerweiterungen

Diese Commits markieren den Übergang von einer Idee zu einer funktionierenden Monitoring-Oberfläche.

## 4. Mai 2026 - SSE-Refactoring
Am `2026-05-04` zeigt die Historie ein SSE-Refactoring. Das ist relevant, weil die aktuelle Produkterfahrung weiterhin stark von Live-Snapshot-Auslieferung statt von manuellem Refresh abhängt.

## 9. Mai 2026 - Repository-Rahmung und README
Commits vom `2026-05-09` fuegten hinzu:

- Mirror-Kontext für das Repository
- Arbeit am öffentlichen README
- Korrekturen beim Bild-Hosting

Das ist der Punkt, an dem `UnyPort` leichter als öffentliches Projekt und nicht nur als lokaler Versuch präsentiert werden konnte.

## 30. Mai 2026 - Packaging-Refactoring
Das Refactoring vom `2026-05-30` ist einer der sichtbar wichtigsten Packaging-Schritte:

- UPX-Kompression für ein kleineres Paket
- Breitere Runtime-Bereinigung
- Stärkere Positionierung als kompaktes Binary

Das passt direkt zum im README wiederholten Single-Binary-Versprechen.

## 6. Juni 2026 - UI und Neustart-Historie
Mehrere Commits vom `2026-06-06` fuegten hinzu:

- Ein Hamburger-Menue
- Eine Git-artige Reboot-Karte
- Ein Netzwerk-Karten-Refactoring

Diese Änderungen bewegen `UnyPort` deutlich in Richtung einer vollstaendigeren Operator-UI statt eines rohen Metrikpanels.

## 7. Juni 2026 - öffentliche Release v0.1.0
Am `2026-06-07` zeigt die Repository-Historie:

- Release-Vorbereitung
- Feinschliff der Release Notes
- Das Tag `v0.1.0`
- Arbeit am App-Versioning

Sowohl `CHANGELOG.md` als auch `RELEASE-v0.1.0.md` bestaetigen, dass `v0.1.0` die erste öffentliche Release ist.

## 18. Juni bis 10. Juli 2026 - Dokumentationsverfeinerung
Spätere Commits konzentrierten sich auf Dokumentation:

- `2026-06-18`: Korrektur der Demo-URL und der Docker-Bind-Dokumentation
- `2026-07-10`: Historien-Update

Bis zum 10. Juli 2026 ist die sichtbare Repository-Geschichte daher die eines monitoring-first Kontrollportals mit öffentlicher Release, kompaktem Packaging und klarerer Dokumentationshaltung.
